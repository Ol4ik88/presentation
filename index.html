MVC (Model-View-Controller)

is a fundamental design pattern whose purpose is to separate user's interface logic from programming logic

The concept of the MVC

  • Model: Manages data and business logic
  • View: Handles layout and display
  • Controller: Routes commands to the model and view parts

Possible difficulties

  • Should some code be assigned to the view or the controller?
  • How much should the controller know?

Data stream

  • passed from the user to the view
  • from the view to the controller
  • the model is updated by the controller
  • the model notifies the view that something has changed

In conclusion

  • The controller determines which view should be displayed at the moment.
  • View events can only affect the controller. The controller can affect the model and define a different view
  • Several views are possible for only one controller
